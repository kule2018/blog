文档流

文档流分为：正常流(static/relative), 绝对定位流(absolute/fixed), 浮动流(float)

  • static
  • relative 0 / 1 / 2 / 3
    相对于自身在 static 状态下所处位置
  • absolute 0 / 1 / 2 / 3 / 4
    相对于祖先中存在 position: relative/aboslute/fixed; 的元素; 不存在时, 相对于 html 元素
  • fixed 0 / 1 / 2 / 3 / 4
    相对于窗口定位
  • float float / clear
  • sticky(only try)

文档流相关属性

  • top/right/bottom/left
  • z-index(require: relative/absolute/fixed)
  • clip: rect( top, right, bottom, left ); // ie6,7: clip: rect(top right bottom left) (require: absolute/fixed)
  • clear - left/right/both
  • overflow - visible/hidden/auto/scroll - 0 / 1 / 2 / 3

外边距(margin)

margin: 元素与父元素或其它元素之间的距离.

对自身和附近元素的影响

  • 在 top 和 left 方向移动, 同时影响附近元素
    • 正值, 向下/右移动
    • 负值, 向上/左移动
  • 在 bottom 和 right 方向, 只影响附近元素
    • 正值, 附近元素向下/右移动
    • 负值, 附近元素向上/左移动(可能出现覆盖现象)

margin与其他属性关系对比

  • position: relative;

    设置 relative + 方向, 元素原位置留白(即相邻元素位置不变), 自身向指定方向移动.

  • position: absolute;

    设置 absolute 属性, 元素脱离文档流, 导致相邻元素做相应移动; 设置方向后, 参照相对元素排版.

外边距合并

相邻元素(兄弟关系或祖先关系)的外边距可能会合并为一个外边距.

合并规则

  • 两外边距都是正数时, 合并结果是其中较大的值
  • 两外边距都是负数时, 合并结果是两者绝对值较大的那个
  • 两外边距一正一负时, 合并结果是两者的相加的结果

合并条件

  • 在同一个 Block Formatting Context 中, 并且是处于静态文档流中的块级(block/table/list-item)元素.
  • 容器和first/last子元素之间, 容器没有 padding/border/clear.
  • 发生在相邻的两个元素的垂直方向上.

现象

  • 非 block 元素不会发生 margin 合并.
  • BFC 内部的元素和外部元素不会发生 margin 合并.
  • 脱离文档流的元素不会发生 margin 合并 - float/absolute/fixed 元素.

Margin 应用

  • Layout
  • 增加元素宽度
  • 模块 Head
  • 隐藏导航分割线
  • 排行榜/分类/ Media/ Media-ext

Block Formatting Contexts

块级格式化上下文, 有自己独立的布局环境和规则, 它不受外部环境影响, 也不会影响外部环境; 可以包裹内部的 float 元素, 同时不与外部 float 元素重叠.

触发条件

  • float: 除 none 以外的值 - 浮动元素
  • position: absolute, fixed - 绝对定位元素
  • display: table-cell(table), table-caption
  • inline-block - 非块级盒子的块级容器
  • display: flex/inline-flex; 弹性盒.
  • overflow: 除了 visible 以外的值( hidden/auto/scroll )

clearfix

.clearfix:before,
.clearfix:after {
  content: '';
  /* 创建 BFC 元素, 隔离 top/bottom 元素, 不让 margin 重叠 */
  display: table;
  /* 其他方法:
    display: block;
    height: 0;
    overflow: hidden;
  */
}
/* 清除浮动 */
.clearfix:after {
  clear: both;
}
/* IE 6/7 */
.clearfix {
  *zoom: 1;
}

Inline-box

box

containing-box/line-box/inline-box(匿名)/content-box

line-box 在 containing-box 中上下排列, 并且它的高度取决于最高的 inline-box.

行高(line-height)与字体尺寸(font-size)的差称为行距; 行距平分后, 分别放在 content-box 上下. 特殊: 当 font-size 大于 line-height 时, 会折叠 line-height 以保证 inline-box 高度(或 content-box 可见).

super/sub 存在的行高会影响 line-box 高度, 细节可设置 line-height: 0; position: relative; /*+top/bottom*/ 修正;

最后, 容器的高度是由 line-height 撑起的(不是 font-size).
